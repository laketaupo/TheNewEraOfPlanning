What is a Transformation Process?

A Transformation Process describes how one or more input items are converted into one or more output items. This is essentially the Bill of Materials (BOM) in Planning software — the recipe that defines production.

Every transformation process has:

  • Inputs — raw materials or components consumed
  • Outputs — finished or semi-finished goods produced
  • Yield — how much output you get per unit of input
  • Lead time — how long the transformation takes
  • Process location — which facility or node performs it

BOM vs. Routing

In traditional ERP, BOM and routing are separate. In Planning software they are unified under the Transformation Process:

Traditional ERP Planning software
BOM (what to use) Transformation input items
Routing (how to make it) Resource consumptions on the process
Work center Resource

Multi-level BOMs

Transformation processes can be chained — the output of one becomes the input of another. Planning software resolves these multi-level structures automatically when generating supply plans.
