## What is a Digital Twin?

A **Digital Twin** is a virtual replica of a physical system that is kept in sync with reality through continuous data feeds. In o9, the Digital Brain *is* your supply chain's digital twin.

Every physical entity — a plant, a machine, a shipment, a pallet of inventory — has a corresponding representation in the graph. When the physical world changes, the digital model updates.

## Digital Twin vs. Static Plan

| Static plan (spreadsheet/ERP) | o9 Digital Twin |
|---|---|
| Snapshot in time | Continuously updated |
| Manual re-planning required | Automatic re-propagation |
| Silos per function (finance, ops, supply) | Single shared model |
| "What was true when we ran the plan" | "What is true right now" |

## What the Twin Enables

1. **Live plan vs. actuals** — see deviations the moment they occur
2. **Exception management** — alerts surface only when action is needed
3. **Scenario simulation** — run "what-if" against the live model without disturbing the baseline
4. **End-to-end visibility** — from raw material supplier to customer shelf in one view

## The Confidence Loop

The digital twin creates a confidence loop:

```
Real world changes
      │
      ▼
Data ingested into Digital Brain
      │
      ▼
Plan updated automatically
      │
      ▼
Planner reviews only exceptions
      │
      ▼
Decision executed back into real world
```
