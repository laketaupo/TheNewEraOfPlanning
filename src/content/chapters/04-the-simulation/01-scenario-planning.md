## What is Scenario Planning?

**Scenario planning** is the practice of running alternative versions of your supply plan to answer *"what if?"* questions — without touching the live baseline plan.

In o9, a **scenario** is a sandboxed copy of the plan where you can:

- Change demand assumptions (e.g., +20% volume in Q3)
- Change supply constraints (e.g., a plant shuts down for 2 weeks)
- Change network parameters (e.g., switch supplier for a key component)
- See the downstream impact instantly

## Baseline vs. Scenarios

| | Baseline Plan | Scenario |
|---|---|---|
| **Purpose** | The agreed, committed plan | An exploratory "what if" |
| **Visibility** | Shared across the organization | Private or shared per team |
| **Impact** | Drives execution (POs, production orders) | Does not drive execution |
| **Lifespan** | Continuously updated | Created, analyzed, then accepted or discarded |

## The Scenario Workflow

1. **Create** a scenario from the baseline (one click)
2. **Change** one or more inputs (demand, supply, constraints)
3. **Run** the planning engine on the scenario
4. **Compare** the scenario output to the baseline
5. **Decide** — accept the scenario as the new baseline, or discard it

## Why It Matters

Traditional planning requires days or weeks to run a new scenario because it involves re-exporting from ERP, re-running spreadsheet models, and re-consolidating results. In o9, a scenario runs in **minutes** on the same live data.
